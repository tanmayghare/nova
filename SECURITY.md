# Security Policy

## Reporting Security Vulnerabilities

We take security seriously. If you discover a security vulnerability in Nova, please report it responsibly.

### How to Report

**Please do not report security vulnerabilities through public GitHub issues.**

Instead, please report them by:
1. Creating a security advisory on GitHub (preferred)
2. Sending an email to the project maintainers

### What to Include

When reporting a security vulnerability, please include:
- Description of the vulnerability
- Steps to reproduce the issue
- Potential impact
- Any suggested fixes or mitigation strategies

### Response Timeline

- We will acknowledge receipt of vulnerability reports within 48 hours
- We will provide a detailed response within 7 days
- We will work to address critical security issues as quickly as possible

### Scope

This security policy applies to the core Nova project repository. Issues in dependencies should be reported to the respective maintainers.

### Experimental Software Notice

Please note that Nova is experimental software in active development. While we take security seriously, this software is not recommended for production environments or handling sensitive data.

## Security Best Practices

When using Nova:
- Always review the code before running it
- Use API keys with minimal necessary permissions
- Be cautious when automating interactions with sensitive websites
- Keep your dependencies updated
- Monitor your usage for unexpected behavior

## Supported Versions

As an experimental project, we currently only support the latest version. We recommend always using the most recent release.
